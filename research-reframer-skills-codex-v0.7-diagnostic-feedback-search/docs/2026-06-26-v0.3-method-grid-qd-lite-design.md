# Research Reframer v0.3 Method-Grid QD-lite Design

Date: 2026-06-26

## Why v0.2 Was Not Enough

The v0.2 workflow made ideation traceable:

```text
system map -> leverage points -> reframes -> idea cards
```

That was already stronger than a single brainstorm prompt, but it remained linear. Once a few leverage points were selected, the workflow could still collapse into the most obvious reframes.

v0.3 adds an explicit exploration layer so the agent can show which parts of the reframing space it explored and which parts remain empty.

## Why Method-Grid QD-lite

The new layer is QD-inspired, not a full MAP-Elites implementation. It borrows three useful ideas:

- define a search space,
- preserve candidates by location in that space,
- select elites without deleting non-elites.

The search space is:

```text
system_leverage_type x lateral_operator
```

This gives each candidate a method cell. A candidate is no longer just "an idea"; it is a grounded attempt to apply a reframing move to a system leverage point.

## Primary Grid

The row axis is `system_leverage_type`:

- `parameter_metric`
- `stock_flow_delay`
- `feedback_information`
- `rule_incentive`
- `self_organization_structure`
- `goal_paradigm`

The column axis is `lateral_operator`:

- `assumption_challenge`
- `reversal`
- `decomposition`
- `analogy`
- `random_stimulus`
- `PO_provocation`
- `entry_point_shift`
- `concept_abstraction`

This grid is more useful than an axis such as "object being reframed" because it describes how the agent is searching, not only what it is searching over.

## Why Target Object Is Metadata

`target_object` is important, but it is not a primary axis. It can vary within the same method cell:

```text
feedback_information x reversal
```

could target attribution, evaluation, explanation, trace editing, or feedback timing. Keeping `target_object` as metadata preserves specificity without exploding the grid.

## Deferred Judgment

`PO_provocation` and `random_stimulus` are intentionally oddness-friendly. They help prevent premature convergence.

The workflow separates:

- divergence: generate grounded candidates, including strange ones,
- evaluation: score candidates with a heuristic rubric,
- archive: preserve candidates and select per-cell elites.

The archive should not delete a candidate simply because it looks strange. It should reject only candidates that cannot be grounded in a real system node and leverage point.

## Why Coverage Matters

Coverage is not a quality guarantee. It is an inspection signal.

If only two cells are occupied, the agent may have generated decent ideas while leaving most method combinations unexplored. If twelve or more cells are occupied, the user can inspect a broader slice of the reframing space.

v0.3 validates:

- total cells,
- occupied cells,
- coverage ratio,
- underexplored cells,
- preserved odd candidates,
- per-cell elites.

## Quality Scores

Each QD candidate receives heuristic scores:

- groundedness,
- frame distance,
- operator fit,
- testability,
- research value.

These scores are agent-generated and editable by humans. They are useful for transparent selection, but they are not proof of novelty, correctness, or research value.

## Connection To Idea Cards

Selected QD candidates become idea cards. v0.3 idea cards add `method_trace`, which links:

```text
input evidence
  -> system node
  -> leverage point
  -> method grid cell
  -> QD candidate
  -> idea card
  -> minimal experiment
```

This preserves the full path from rough input to testable research direction.

## Static Visualization

The first visualization is `04_method_grid.html`, generated by `scripts/render_method_grid.py`.

It is intentionally static:

- no React,
- no Next.js,
- no Streamlit,
- no database,
- no external CDN.

The HTML shows coverage metrics, the 6x8 grid, candidate counts, elites, scores, preserved odd candidates, and underexplored cells.

## Limitations

v0.3 does not claim:

- full MAP-Elites,
- objective novelty evaluation,
- automatic research-quality guarantees,
- replacement of human judgment,
- literature-review completeness.

The value is traceable reframing and searchable method coverage.

## Future Work

- Interactive UI for filtering, comparing, and editing candidates.
- Memory or preference learning across workshop runs.
- A `vertical-refine` skill for deepening selected idea cards.
- Baseline evaluation against direct brainstorming.
- Human feedback loops for adjusting heuristic scores.

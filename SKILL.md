# frameshift —— 问题空间上的价值地图

输入一篇论文，frameshift 不去总结它做了什么，而是去找它**没做、且被默认锁死**的选择
（被夹持的自由度），用水平思考算子把它们松开，变出新的研究问题。每个问题先过证伪闸门，
过了才上地图，按"商业行动性 × 概念深度"铺成一张二维价值地形。

定位是它的核心立场：**机器把价值地图铺出来，人来挑哪几个值得做、并把选中的丢回去再定向。**
这是一个人机协作工具，不是自动选题机——agent 跑四个工位的回路，人在地图上做决策。
零 API key、零外部依赖：你（执行本技能的 agent）扮演四个工位，确定性的部分交给 `engine`。

## 触发
`/frameshift <论文路径或描述> [--run <名字>]`

## 前置
- 在仓库根目录运行；Python 3（标准库，无第三方依赖）。
- 看板（可选）：`cd dashboard && npm install && npm run build`（一次即可）。

## 四工位回路

回路是 **定位 → 发散 → 收敛/鉴价 → 价值地图**，角色卡在 `roles/`，逐张照着做：

1. **工位一 · 定位**（`roles/locate.md`）
   读论文，找出被夹持住的自由度，写 `runs/<id>/graph.json`：
   `paper` / `original_question` / `dominant_idea` / `clamped_dofs[]`（每条带杠杆层级
   L1参数·L2结构·L3规则信息·L4目标范式、为什么被夹、松开会打开什么）。
   优先找 L3/L4 的夹持——盲区与价值都在那里。

2. **工位二 · 发散**（`roles/diverge.md`）
   挑一条自由度，选一个算子（挑战假设/反转/换行动者/分解/类比/随机刺激/PO/生成替代/去约束），
   产出候选卡草稿 `runs/<id>/cards/<id>.json`，只填 `id/parent_dof/operator/question`。
   ⚠️ 这一步**禁止任何合理性评判**（暂缓判断）。轮换使用算子，一条夹持多产几个问法。

3. **工位三 · 收敛 / 鉴价**（`roles/appraise.md`）
   先过**证伪闸门** `floor`（能否被最小实验证伪？过不了直接 `status:rejected`），
   过了再填 `value_profile`（商业行动性/概念深度/可落地性/对谁有价值）与
   `prior_art`（最接近的已知工作、新颖水印、新颖度）。
   ⚠️ 发散与鉴价必须是两个独立步骤，不得边生成边评判。

4. **工位四 · 价值地图**（`roles/value_map.md`）
   运行 `python -m engine.cli assemble --run <id>`：引擎按固定映射
   （x=商业行动性，y=概念深度，{low:0.15,mid:0.5,high:0.85}；点大小=可落地性；
   角标=新颖度；原问题钉在增量改进角 (0.15,0.15)）确定性地铺出 `map.json` / `run.json`。
   向用户讲清地形，请用户挑 1–3 个点**再定向**：选中的可作父卡丢回工位二继续变异（记 lineage），
   或要工位一去松开相邻的另一条自由度。

## 数据放哪
所有运行数据在 `runs/<run_id>/`：
- `graph.json` —— 工位一产出（论文 + 被夹持的自由度）
- `cards/<id>.json` —— 每张候选卡（工位二+三逐步填充）
- `map.json` / `run.json` —— 工位四由 `assemble` 确定性生成（勿手改）

## 看地图
后台跑 `python -m engine.cli serve`，浏览器开 `http://127.0.0.1:8420`：
横轴商业行动性、纵轴概念深度，点大小=可落地性、角标=新颖度、◎=原问题。
点选任一点看完整推导链（松开了哪条自由度→用了什么算子→证伪闸门→价值画像→先验技艺→谱系）。

## 红线
- 不得跳过角色卡直接发挥；不得为凑地图好看而虚标坐标或分数。
- 卡片 JSON 是唯一真相；`map.json`/`run.json` 由 `assemble` 生成，要改图就改卡再重跑。
- 本技能只产出研究问题与实验设计，不执行实验（实验交给下游工作流）。
- 人是回路里挑选与再定向的一环——不要替用户拍板该做哪个方向。

# Research Reframer Skills

**Version 0.5.0** · see [CHANGELOG.md](CHANGELOG.md). v0.5 is a contract-breaking redesign: the
two-gate method-grid flow becomes a **three-gate cognitive pipeline** — *systems thinking → lateral
thinking → vertical thinking* — with an adversarial, dual-judge audit. `schema_version` is now `2.0`;
v0.2/v0.3 artifacts do not validate under it. See
[the v0.5 design doc](docs/2026-06-27-v0.5-three-gate-redesign.md).

Research Reframer is a Claude Code / Codex-compatible skill pack for early-stage research ideation.

Most research agents help users search, summarize, write, or optimize within a given problem frame.
Research Reframer targets an earlier bottleneck: **reframing the problem itself**, and keeping the
human in control of the three decisions that matter.

## The three gates

The pipeline mirrors de Bono's split between *generating* and *selecting*, plus the systems-thinking
question of *where to intervene*:

```text
input (rough idea / paper gap / project note)
  → system-map        01_system_map.json
  → leverage-scan     02_leverage_points.json        (LP-### : Meadows-style leverage points)
  ── Gate 1 · systems : human picks LEVERAGE POINTS — "where in the system to cut" ──
  → lateral-generate  03_lateral_reframes.json       (LR-### : diverge; operators internal; judgment DEFERRED)
  ── Gate 2 · lateral : human picks LATERAL SCHEMES — "which divergent moves are interesting" ──
  → vertical-audit    04_vertical_audits.json        (VA-### : adversarial, default-reject, DUAL-judge)
  ── Gate 3 · vertical: human picks AUDITED SCHEMES — "which survive scrutiny" ──
  → idea-card         06_idea_cards.json             (IC-### : traceable, testable cards)
```

Each card is traceable end-to-end: `input_evidence → system_node → LP → LR → VA → IC`, and the
validator machine-checks that chain.

### Why three gates, not two

- **Gate 1 selects leverage points, not method cells.** The human chooses a *place in the system*
  (`LP-007: the trace/evaluation feedback loop`), not an abstract `type × operator` cell. Lateral
  operators become an internal generation mechanism.
- **Generation and judgment are separated.** `lateral-generate` stays generative and defers judgment
  (schemes carry no score, `not_yet_audited: true`). `vertical-audit` is a *separate*, adversarial
  convergence stage.
- **The audit is adversarial and dual-judge.** Each Gate-2 scheme is judged independently by an
  **external** engine (Codex) and Claude, both default-reject. Agreement is taken; disagreement is
  escalated to the human at Gate 3. Rejected audits are kept with reasons — the "rejected drawer" is
  part of the value, not hidden.

## Install

Install all skills into the user-level Claude Code and Codex skill folders:

```powershell
python scripts/install_skills.py --target both
```

This maps to `Claude Code: $HOME/.claude/skills` and `Codex: $HOME/.agents/skills`. Install only one
engine with `--target claude-user` or `--target codex-user`; install repo-scoped copies with
`--target claude-repo --repo-root .` (or `codex-repo` / `both-repo`). By default the installer refuses
to overwrite an existing install — add `--force` to replace one. Dry run first:

```powershell
python scripts/install_skills.py --target both --dry-run
python scripts/install_skills.py --target both --force
```

Restart Claude Code or Codex so the new skills load.

For distribution, build and smoke-test the portable zip:

```powershell
python scripts/package_zip.py
python scripts/smoke_test_zip.py dist/research-reframer-skills-pack.zip
```

## Quickstart

Run the workshop on the bundled example:

```text
Use $reframe-workshop on examples/physmaster-three-gate/input.md.
Write markdown + json outputs into examples/physmaster-three-gate/outputs-new.
Run the three-gate workflow. At each gate, either let me choose (and log it) or pause for my selection.
For the vertical audit, use Codex as the external judge if available, else self-audit honestly.
```

Expected artifact layout (the `.json` files and the audit trail are required; per-artifact `.md`
summaries and the static `.html` views are optional, produced on request / by `render_outputs.py`):

```text
outputs/
  01_system_map.json
  02_leverage_points.json
  03_lateral_reframes.json        # + 03_lateral_reframes.html  (render_outputs.py)
  04_vertical_audits.json         # + 04_vertical_audits.html   (render_outputs.py)
  05_human_selection.md           # required: the Gate 1/2/3 selections
  06_idea_cards.json
  decision_log.md                 # required: per-stage decisions + the dual-judge table
  reframe_report.md               # recommended: human-readable summary
```

The bundled examples ship the `.json` + the audit trail + the rendered `.html`; readable `.md`
companions for `01/02/03/04/06` are optional and not generated by default.

## Skills

- `system-map`: convert rough research text into actors, goals, stocks, flows, rules, information
  flows, boundaries, failure modes, and uncertainties.
- `leverage-scan`: identify high-leverage places (`LP-###`) where the research frame can change.
- `lateral-generate`: diverge from the Gate-1 leverage points into lateral schemes (`LR-###`),
  sweeping the 8 lateral operators internally; judgment deferred; diversity kept via a coverage ledger.
- `vertical-audit`: adversarially audit the Gate-2 schemes (`VA-###`) with dual judges (Codex + Claude),
  default-reject; escalate disagreements to the human.
- `idea-card`: convert Gate-3-selected audits into traceable, testable cards (`IC-###`).
- `reframe-workshop`: orchestrate the full three-gate, human-in-the-loop workflow.

## External judge (Codex)

`vertical-audit` can call Codex as an independent, external auditor so the judge is not the model that
generated the scheme:

```sh
codex exec --skip-git-repo-check -s read-only \
  --output-schema <verdict.schema.json> -o <out.json> - < <prompt.txt>
```

Read-only sandbox is sufficient (the auditor only reads). If `codex` is unavailable, Claude self-audits
under the same rubric and records `auditor: "self"` for honest provenance. Both judges are LLMs — an
external judge is more defensible than self-judgment but is **not** ground truth.

## Interactive gates (clickable, zero-setup — the default)

Every human gate is presented as an **inline clickable panel**: the candidates (`LP-###` / `LR-###` /
`VA-###`) render as toggle rows with a "let Claude decide" button, and a click flows straight into the
conversation to drive the next stage. It uses Claude Code's built-in widget capability
(`mcp__visualize__show_widget` + `sendPrompt`) with an `AskUserQuestion` fallback, so it needs **no
server, no launch flag, and no per-use setup** — install the pack once and every gate is one click.
Gate 3 also shows the both-judges-rejected audits in a read-only "rejected drawer". The panel template
ships with the pack: [`skills/reframe-workshop/references/gate_widget.md`](skills/reframe-workshop/references/gate_widget.md).

`reframe-ui/` is a separate **optional, advanced** browser channel (research preview) for those who want
a standalone localhost UI. It is **not** required and **not** the default: custom Claude Code channels
must be launched with `--dangerously-load-development-channels` *every* time plus a local server. See
[`reframe-ui/README.md`](reframe-ui/README.md).

## Validate

The validator is stdlib-only and checks the v0.5 contract + the full trace chain:

```powershell
python scripts/validate_outputs.py examples/physmaster-three-gate/outputs
```

It enforces required fields, enums, `LP-###/LR-###/VA-###/IC-###` id formats, the coverage-ledger
arithmetic, dual-judge consistency, audit-score means, the `LP → LR → VA → IC` field-equality, and the
existence of the human-gate audit trail. The installed `reframe-workshop` skill carries the same
validator and its `contract.py` so installed copies validate standalone:

```powershell
python $HOME\.claude\skills\reframe-workshop\scripts\validate_outputs.py path\to\outputs
```

Before publishing a pack, check that the bundled validator/contract/schemas match the root assets:

```powershell
python scripts/check_bundle_consistency.py
python scripts/package_zip.py
python scripts/smoke_test_zip.py dist/research-reframer-skills-pack.zip
```

## Contract validation vs quality scoring

Contract validation checks **structure and trace**: required fields, enum values and id formats,
cross-artifact trace integrity, coverage-ledger arithmetic, dual-judge consistency, audit-score means,
and idea-card → audit field equality. It also emits non-failing warnings for diversity collapse and
ungrounded / generic rationales.

Quality scoring is different: heuristic, agent-generated, human-editable, and **not** a proof of
novelty, correctness, or research value. The audit's default-reject posture produces real, reasoned
rejections — but the verdict of an LLM judge is an opinion, not ground truth.

## Before / After

Before:

```text
A rough note about an autonomous AI research agent that may look like prompt engineering.
```

After:

```text
1. System map: actors, goals, rules, information flows, feedback loops, boundaries, failure modes.
2. Leverage points: where the frame can change (metrics, the trace/eval loop, who judges value, the
   human's role, failed-path memory).
3. Gate 1: the human picks leverage points to explore.
4. Lateral schemes: diverse divergent reframes per leverage point (judgment deferred), with a coverage
   ledger over LP × operator.
5. Gate 2: the human picks the interesting schemes.
6. Vertical audits: dual adversarial judges (Codex + Claude), default-reject; survivors, escalations,
   and a non-empty rejected drawer with reasons.
7. Gate 3: the human picks audited survivors.
8. Idea cards: selected audits become traceable minimal experiments with failure cases — full
   LP → LR → VA → IC lineage.
```

## Examples

Two end-to-end v0.5 demos, each with a real Codex + Claude dual-judge audit and a full validated
`outputs/` set (system map → leverage points → lateral schemes → vertical audits → idea cards, plus
the audit trail and static `03/04` HTML):

- `examples/physmaster-three-gate`: an autonomous AI-physicist paper gap (PHYSMASTER).
- `examples/ofamas-three-gate`: a one-for-all multi-agent topology-design paper gap (OFA-MAS).

In both, Codex (the external judge) rejected every Gate-2 scheme; the human rescued the two that
survived as sharp empirical falsifications, and the two agreement-rejects sit in the "rejected drawer"
with reasons — a non-empty true-negative set the v0.3 pipeline never produced.

## Method inspiration

Research Reframer uses self-contained method abstractions:

- **Systems thinking** (where to intervene): actors, goals, stocks, flows, feedback loops, rules,
  information flows, delays, boundaries, failure modes — and Meadows-style leverage points.
- **Lateral thinking** (generate, defer judgment): assumption challenge, reversal, decomposition,
  analogy, random stimulus, PO provocation, entry-point shift, concept abstraction.
- **Vertical thinking** (select, deepen, refute): an adversarial default-reject audit with a falsifiable
  minimal experiment, discriminability from prior work, and a so-what test.

The repository intentionally does not include book PDFs or large quoted excerpts.

## Project shape

```text
research-reframer-skills/
  skills/        # system-map, leverage-scan, lateral-generate, vertical-audit, idea-card, reframe-workshop
  schemas/       # JSON Schemas (data contracts) mirroring scripts/contract.py
  scripts/       # contract.py (source of truth), validate / install / package / smoke / bundle-check
  examples/      # physmaster-three-gate end-to-end demo
  docs/          # design docs (v0.2 → v0.5)
  reframe-ui/    # optional Claude Code channel: clickable UI for the gates
  CHANGELOG.md
```

`scripts/contract.py` is the single source of truth for the vocabulary and id/version contract; the
schemas mirror it and the validator imports it. `reframe-workshop` bundles the validator, `contract.py`,
and the schema references so an installed copy validates standalone; `check_bundle_consistency.py` keeps
the bundled copies byte-identical to the root.

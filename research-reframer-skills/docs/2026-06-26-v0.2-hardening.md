# Research Reframer v0.2 Hardening Notes

## Goal

Make the v0.1 skill pack easier to install, distribute, and verify without
turning it into a standalone CLI or web workbench.

## Scope

This pass focuses on:

- Codex and Claude Code target paths.
- Repo-scoped and user-scoped installs.
- A self-contained validator inside the installed `reframe-workshop` skill.
- README quickstart steps that a new user can run in minutes.

This pass intentionally does not add:

- A standalone runner or UI.
- Persistent preference memory.
- Research-quality scoring.
- A new `vertical-refine` skill.

Those belong in later versions after the installable skill pack is stable.

## Target Layouts

```text
Codex user:      $HOME/.agents/skills
Codex repo:      $REPO_ROOT/.agents/skills
Claude user:     $HOME/.claude/skills
Claude repo:     $REPO_ROOT/.claude/skills
Legacy Codex:    $HOME/.codex/skills
```

The installer keeps `legacy-codex` for older local setups, but `codex` now means
the official user-level `$HOME/.agents/skills` target.

Repo-scoped installs require an explicit `--repo-root` so running the installer
from an arbitrary current working directory cannot write skills into the wrong
project.

`--force` only overwrites a target folder when the installed `SKILL.md` name
matches the source skill name. It refuses to overwrite symlinked targets.

## Validation Boundary

The validator is a contract checker. It verifies artifact shape and trace
references. It does not claim to evaluate novelty, feasibility, or research
quality.

Installed copies of `reframe-workshop` include:

```text
scripts/validate_outputs.py
references/*.schema.json
references/contract.md
```

This makes validation available even when the user only installs the skill
folders and does not keep the repository root.

Run this consistency check before publishing:

```powershell
python scripts/check_bundle_consistency.py
```
